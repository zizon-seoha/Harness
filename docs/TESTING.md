# Testing

## 검증 게이트

각 task 구현 후, 변경 범위에 맞는 검증 명령을 직접 실행한다. **이 명령들이 통과해야 task를 `완료`로 본다.**

명령 목록은 `docs/STACK.md`의 「명령어」 절이 단일 출처다. 변경 범위에 맞는 것만 실행한다.

- 백엔드 코드(`*.py`) 변경 → `docs/STACK.md` 백엔드 명령 (ruff / mypy / pytest)
- 프론트(`web/`) 변경 → `docs/STACK.md` 프론트 명령 (lint / tsc / vitest / build)

순서:

1. 관련 코드 수정 완료 확인
2. 변경 범위에 맞는 STACK.md 명령 실행
3. 실패 시 다음 단계로 넘어가지 않고 수정. 통과할 때까지 반복
4. 현재 task의 구현 범위·완료 조건과 일치 확인
5. `docs/PLANS.md` 상태 갱신

## 사람 확인 (재정의)

Claude가 명령으로 직접 검증할 수 없는 **런타임·통합 항목만** 사람 확인 대상이다. 구체 항목은 `docs/STACK.md`의 「도메인 / 사람 확인 항목」 절을 인용한다.

- 사람 확인이 필요한 항목은 사용자에게 명확히 요청하고, 확인 응답 전에는 해당 항목을 완료 처리하지 않는다.
- 자동 명령으로 검증 가능한 것은 사람 확인으로 미루지 않는다.

## 수동 검증 기준

- 사람 확인 항목은 현재 구현된 범위만 대상으로 한다.
- 아직 연결되지 않은 자동 흐름이나 다음 단계 전제를 검증 항목에 섞지 않는다.
- 실패 시 어떤 현상을 기록해야 하는지 분명해야 한다.
