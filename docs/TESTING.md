# 검증 게이트 — 소마 코드배틀

CLAUDE.md에서 `@docs/testing.md`로 참조. "코드 변경 후 무엇을 통과시켜야 하나"의 단일 출처.

## 원칙
- 변경 범위에 해당하는 게이트는 **반드시** 통과시킨 뒤 완료 보고한다.
- 게이트를 못 돌렸으면 "안 돌림"이라고 명시한다(통과로 가장하지 않음).

## 현재 게이트 (셋업 진행에 따라 갱신)
- 프론트(React+Vite+TS): `npm run build`(= `tsc` 타입체크 포함) 통과.
  - lint(`eslint`)·test(`vitest`)는 셋업 시 추가하고 여기 갱신. 추가 전까지 "없음".
- 백(Spring): `./gradlew build`(컴파일+테스트) 통과.
  - 단위 테스트(`./gradlew test`)는 코드 작성과 함께 추가. 추가 전까지 "없음".

## 결정화(권장)
- 게이트가 확정되면 `.claude/settings.json` 훅(PreToolUse/Stop) 또는 CI로 박아 누락을 막는다.
- 산문 "검증해라"에 의존하지 않는다.

## 사람 확인 (Claude가 검증 불가)
- 런타임/통합 항목은 CLAUDE.md "사람 확인" 절 참조 (생성 실패 알림, 마감 10분 전 알림, 외부 키 제공).
