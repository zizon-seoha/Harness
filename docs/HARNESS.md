# Harness

## 목적

이 문서는 AI 에이전트 작업을 문서, task, 상태, 검증, 사람 확인 절차로 통제 가능하게 유지하기 위한 기준입니다.

## 원칙

- 새 규칙이나 제약을 추가하기 전에 기존 문서에 같은 책임이 있는지 확인합니다.
- 같은 의미의 규칙을 여러 문서에 중복 작성하지 않습니다.
- 공통 규칙은 책임 문서에 한 번만 둡니다.
- task 문서에는 현재 task의 구체 산출물, 범위, 수동 작업, 사람 확인 절차만 적습니다.
- 하네스 이탈이 발생하면 금지 문구를 반복 추가하지 않고 실패한 절차 지점을 먼저 찾습니다.

## 규칙 추가 판단

새 규칙이나 제약을 추가하기 전에는 아래 순서로 판단합니다.

1. 기존 문서 중 어느 문서가 이 책임을 가져야 하는지 확인합니다.
2. 기존 문서에 넣으면 책임이 섞이거나 문서가 비대해지는지 확인합니다.
3. 여러 문서에 걸친 운영 기준이면 독립 문서가 더 적합한지 확인합니다.
4. 새 컨텍스트가 반드시 알아야 하는 기준이면 `AGENTS.md` 또는 먼저 읽을 문서 목록에서 참조되게 합니다.

## 하네스 이탈 대응

에이전트 작업 흐름 이탈이 발생하면 아래 순서로 판단합니다.

1. 어떤 절차가 지켜지지 않았는지 확인합니다.
2. 기존 문서에 이미 해당 규칙이 있는지 확인합니다.
3. 있다면 문서를 늘리지 말고 해당 규칙이 작업 흐름에서 확인되도록 조정합니다.
4. 없다면 책임 문서를 하나만 정해 최소한으로 추가합니다.
5. 수정 후 관련 문서끼리 중복되거나 충돌하지 않는지 확인합니다.
