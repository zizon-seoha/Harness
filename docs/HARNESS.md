# Harness

## 목적

이 문서는 에이전트 작업을 문서, task, 상태, 검증으로 통제 가능하게 유지하기 위한 기준이다.

## 원칙

- 새 규칙이나 제약을 추가하기 전에 기존 문서에 같은 책임이 있는지 확인한다.
- 같은 의미의 규칙을 여러 문서에 중복 작성하지 않는다.
- 공통 규칙은 책임 문서에 한 번만 둔다.
- task 문서에는 현재 task의 구체 산출물, 범위, 사람 확인 절차만 적는다.
- 이탈이 발생하면 금지 문구를 반복 추가하지 않고 실패한 절차 지점을 먼저 찾는다.

## 규칙 추가 판단

1. 어느 문서가 이 책임을 가져야 하는지 확인한다.
2. 기존 문서에 넣으면 책임이 섞이거나 문서가 비대해지는지 확인한다.
3. 여러 문서에 걸친 운영 기준이면 독립 문서가 더 적합한지 확인한다.
4. 새 컨텍스트가 반드시 알아야 하는 기준이면 `CLAUDE.md` 문서맵에서 참조되게 한다.
5. 코딩 규칙이면 보편 원칙은 `docs/STYLEGUIDE.md`, 언어별 금지패턴은 `.claude/rules/`에 둔다.

## 이탈 대응

작업 흐름 이탈이 발생하면 아래 순서로 판단한다.

1. 어떤 절차가 지켜지지 않았는지 확인한다.
2. 기존 문서에 이미 해당 규칙이 있는지 확인한다.
3. 있다면 문서를 늘리지 말고 그 규칙이 작업 흐름에서 확인되도록 조정한다.
4. 없다면 책임 문서를 하나만 정해 최소한으로 추가한다.
5. 수정 후 관련 문서끼리 중복·충돌하지 않는지 확인한다.

## 네이티브 기능 우선

같은 통제를 Claude Code 네이티브 기능으로 할 수 있으면 prose 규칙보다 그것을 쓴다.

- 변경 전 승인 → Plan Mode + `.claude/settings.json` 권한 allowlist
- 경로별 코딩 규칙 → `.claude/rules/*.md` (`paths:` 글롭 스코프)
- 반복 작업 → `.claude/commands/*.md` 슬래시 커맨드
- 작업 이력·되돌리기 → git 커밋 + `/rewind`
