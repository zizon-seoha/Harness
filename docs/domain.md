# 도메인 상세 — 소마 코드배틀

CLAUDE.md에서 `@docs/domain.md`로 참조. 실수 방지용 비즈니스 규칙 단일 출처.

## 문제 출제 스케줄
- 월/수/금 0시 공개, 화/목/토 밤 24시 마감. 일요일은 출제 없이 랭킹·피드백 열람만.
- 회차: 월화=1, 수목=2, 금토=3.
- 매 회차 9문제 = Python/C/Java × Easy/Normal/Hard 각 1개.
- 전날 밤 23시 스케줄러가 다음날 회차 자동 생성.
- 마감 10분 전 알림은 scheduler에서 처리.

## 난이도 점수
- Easy: 기본 30.00 + AI 30.00 = 최대 60.00
- Normal: 기본 50.00 + AI 30.00 = 최대 80.00
- Hard: 기본 70.00 + AI 30.00 = 최대 100.00
- 모든 점수 소수점 2자리. 같은 문제는 마지막 제출만 반영.

## AI 피드백 점수 항목 (각 최대 10.00)
- 시간복잡도 효율 / 코드 가독성 / 풀이 독창성.

## AI 문제 생성 3단계 필터
1. 유사도 검사 — 기존 문제와 80% 이상 유사 시 재생성.
2. 정답 검증 — AI 정답 코드 생성 → Judge0 실행해 예제 통과 확인(예제 출력 확정).
3. 난이도 재확인 — AI에게 난이도 자문(운영상 낮춰 허용, 로그만).
- 3회 재시도 실패 시 관리자 알림.

## 랭킹 집계
- 별도 테이블 없이 `submissions.submitted_at` 기준 필터. (user, problem) 마지막 제출만 합산.
- 기간: 2일(회차 공개~마감) / 주간(월 0시~일 24시) / 월간(1일~말일).
- 분류: 전교 통합 / 학년별 / 반 대항전(평균) / 반 내 개인.
- 월간 랭킹은 4번째 주 일요일에만 공개, 그 외 잠금 + D-day.
- 일요일/4번째 일요일 판정: 프론트 `dateUtils`(isSunday/isFourthSunday), 백엔드는 동등 유틸(Spring).

## 주간 주제 순환 (8주)
동적 프로그래밍 → 그래프 탐색(BFS/DFS) → 정렬 → 스택/큐 → 재귀 → 그리디 → 이진 탐색 → 문자열 처리.

## 외부 API
- OpenAI: 문제 생성 + 피드백(즉시 / 2일 / 주간 / 월간). 모델은 환경변수 `OPENAI_MODEL_*` (CLAUDE.md 포인터 참조).
- Judge0: 코드 실행/채점. 언어 ID Python=71 / C=50 / Java=62.
